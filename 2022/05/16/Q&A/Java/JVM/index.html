JVM

JVM组成部分及作用

组成部分 作用
类加载器（classLoader） 将编译好的java字节码（class文件）加载到内存中
运行时数据区（Runtime Data Area） 用来存放java字节码文件
执行引擎（Execution Engine） 将字节码翻译成底层系统指令，即将jvm指令集翻译成操作系统指令集，可以理解为java字节码解释器，然后交给cpu去执行
本地库接口（Native Interface） 字节码翻译成底层系统指令的过程中会调用一些功能接口，比如驱动，这些接口即本地库接口

线程私有 创建线程时创建，线程退出时销毁 方法区、堆

什么是双亲委派模型

类加载时，类加载器会先委派给父类加载器完成加载动作，每一层的都是如此，当父类的类加载器无法完成时，子类加载器才会尝试加载类

类加载执行过程

1、加载：根据查找路径找到class文件并导入
2、检查：检查已加载class文件正确性
3、准备：给类中的静态变量分配空间
4、解析：jvm将符号引用替换成直接引用，即将标示替换为指向真实的内存地址
5、初始化：对静态变量和静态代码块执行初始化工作

如何判断对象可以回收

判断方式 判断逻辑
引用计数器 当引用计数器值为0时，可以回收，该方式会存在循环引用问题
可达性分析 当一个对象到GC roots都没有任何引用链相连时，可以回收

java中有哪些引用类型

引用类型 简介
强引用（Strong Reference） 默认引用方式，只要强引用在，强引用对象内存空间就不会被回收
软引用（Soft Reference） 只有在内存不足时，软引用对象才会被回收
软引用（Weak Reference） 只要垃圾回收执行，弱引用对象就会被回收
虚引用（幽灵引用/幻影引用） 跟踪垃圾回收活动，当GC检测到虚引用对象时，会将该引用放到ReferenceQueue中，由程序员自己调用ReferenceQueue.pull()移除引用之后，GC才会回收该对象占用内存，即回收由程序员控制

JVM垃圾回收算法

算法 简介 优点 缺点
标记-清除算法（Mark-Sweep） 先对存活对象进行标记，然后清除未被标记对象 实现简单，不需要移动对象 内存碎片化严重
标记-整理算法（Mark-Compact） 先标记存活对象，然后将存活对象移动到内存一端，然后将端边界外的内存回收 解决了标记-清理算法的内存碎片化问题，相对复制算法更节约内存 仍需要移动对象，一定程度上降低了效率
复制算法（Copying） 将内存一分为二，当垃圾回收时，将存活对象复制到另一半内存中，然后清理当前一般内存 按顺序分配内存即可，实现简单，运行高效，不用考虑内存碎片 可用内存缩小，对象存活率高时需要频繁复制
分带算法 大部分JVM采取方案，根据对象生命周期划分到不同内存区域，一般分为新生代（每次垃圾回收时有大量垃圾需要回收）、老生代（每次垃圾回收时只有少量对象被回收），新生代采取复制算法，老生代采用Mark-Compact算法 目前问题最少 实现复杂

JVM垃圾回收器

垃圾回收器 简介
Serial 最早的单线程串行垃圾回收器
Serial Old Serial垃圾回收器的老年版本，同样是单线程，可以作为CMS垃圾回收器的备选方案
ParNew 是Serial的多线程版本
Parallel 类似ParNew是多线程，但吞吐量优先，可以牺牲等待时间换取吞吐量，采用复制算法进行垃圾回收
Parallel Old Parallel的老生代版本，使用标记-整理算法进行垃圾回收
CMS 以获得最短停顿时间为目标的垃圾回收器，适用B/S（（Browser/Server，浏览器/服务器模式））系统
G1 一种兼顾吞吐量和停顿时间的GC实现，JDK9之后的默认GC选项

详细介绍CMS垃圾回收器
全称 Concurrent Mark-Sweep
特点 以牺牲吞吐量来获得最短停顿时间
适用场景 要求响应速度的服务应用
开启方式 启动JVM参数中加上”-XX:+UseConcMarkSweepGC”来指定使用CMS
使用垃圾回收算法 标记-清除
缺点 由于使用标记-清除算法，所以会产生大量内存碎片
剩余内存不足处理逻辑 系统会出现Concurrent Mode Failure。临时CMS会采用Serial Old回收器完成垃圾回收，此时性能会降低

新生代垃圾回收器和老生代垃圾回收器有哪些，区别

新生代回收器 老生代回收器 永久代
对应垃圾回收器 Serial、ParNew、Parallel Scavenge Serial Old、Parallel Old、CMS G1
采用算法 复制算法 标记-整理算法

分代垃圾回收器如何工作

分区

  • 新生代（默认占用1/3内存空间）
    • Eden(默认占用8/10内存空间)
    • To Survivor(默认占用1/10内存空间)
    • From Survivor(默认占用1/10内存空间)
  • 老生代（默认占用2/3内存空间）

    新生代分区工作逻辑

    1、将Eden和From Survivor中存活的对象放到To Survivor中
    2、清空Eden和From Survivor分区
    3、To Survivor和From Survivor相互切换，新生代分区采用复制算法，所以二者会切换

    老生代分区工作逻辑

    1、当老生代占用空间达到某个值时会触发垃圾回收，一般使用标记-整理算法
    2、每当在From Survivor和To Survivor移动时都存活的对象，年龄就+1，当年龄达到15（默认值15）时，升级为老年代。大对象会直接进入老年代

参数名称 介绍

-Xmx2g 堆最大内存为2g

-XX:PrintGC 开启打印GC信息
-XX:+PrintGCDetails 打印GC详细信息
