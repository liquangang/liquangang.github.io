## 会议原则：
```
* 能不开会就不开会，根据开会目的与目标来确定
* 分清自己的角色，做好会前准备和会议引导
```
## 会议过程：
#### 1. 会前准备
* 会议召集人会前准备工作: 

```
* why：开会是否能解决问题，是否需要开会
    * 需要多人（n>2）一起解决的问题
    * 邮件无法描述清楚
    * 比较紧急需要决定的事情
    * 需要更多人当面沟通效果更好的事情
```
```
* who：需要谁参加
    * 不知道谁参加时：请求经理、技术指导人
    * 控制不超过8人：参加会议的人必须是能够对会议目标作出贡献的人
    * 设计会议程序，部分人可以晚来或者先走
    * 将大部分会议分成几个小的会议
    * 对于部分与会者，可提前征求意见，减少不必要的参与
```
```
* time：需要开多久
    * 鼓励快速的短会
    * 会议时间1小时以内
    * 如果会议有延长的可能性，考虑安排在11点或者17点
```
```
* when&where：会议开始时间&地点，并发出会议通知
    * 描述清楚目标、希望解决的问题
    * 时间、地点、参与人列表
    * 参与人需要做的准备工作
    * 预计持续时间
```
* 会议参与人会前准备：

```
* why：是否需要参加
* when：会议时自己的时间安排是否合适
```
#### 2. 会议引导
```
* 召集人：将自己看作服务者
* 预先对议题有一个整体研究（过去、现状、预期）
* 能够聆听他人发言，给予意见不同者一定的反对时间，鼓励发言者进一步阐明观点或留以后研究
* 纠正误解和不相关的话题
* 有下列情况应立即结束讨论
    * 意见已经达成一致
    * 进一步讨论的资料缺乏
    * 讨论所需的人没有参加会议
    * 成员认为需要时间考虑或者下来商量
    * 讨论说明原有议题的假设已经发生改变
    * 本次会议太长，没有时间更好的讨论该议题
    * 有反对意见，达成一致较困难
    * 结束议题后应该总结一下
```
#### 3. 会议总结
```
* 会议之后进行会议总结
* 如果需要，向参会人发出会议总结邮件
```
